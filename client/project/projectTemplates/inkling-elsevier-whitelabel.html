Whitelabeling the Inkling Store

Inkling partnered with Elsevier, an international textbook company that has a large set of medical resources, specifically working with their products Expert Consult and Student Consult. These two products were already existing websites where users could access PDF versions of their print books. Elsevier wanted to upgrade these users’ experience by bringing them over to the Inkling platform. However they wanted also to maintain a separate environment for these users, Expert and Student Consult stores and readers powered by Inkling. The solution they concluded upon was to whitelabel our Inkling Store and reader.

Using our Inkling Store code base, we set up separate instances at expertconsult.inkling.com and studentconsult.inkling.com. These all fed from the same front end and back end, with some minor changes to the code to enable what are essentially sibling sites. This meant that any major design changes to the store had to be considered across all three instances of the store.

As part of this process, I explored how to translate our new visual language to incorporate another brand. This meant balancing maintaining Inkling’s brand identity with making sites unique to Expert and Student Consult. In this final implementation (above), there is a strong connection made to the Inkling web store through base colors and typography, but accent colors and images highlight that this is in fact an Expert Consult store.
