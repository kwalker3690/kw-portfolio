Setting Up Sass Files

September 23, 2014

Sass is an amazing CSS preprocessor. It lets you write CSS quickly and in an amazingly modular fashion. It helps you cut down the amount of CSS you have to write by letting you make variables and even functions to come up with reusable CSS. It's super easy to learn, I highly recommend checking out their introduction to get started.

Now that you've got the basics covered, let's talk about setting up your Sass files. Sass's core benefit is their modularity, and the file structure should reflect that modularity. I'll go over a few different possibilities of how to set up your files, depending on the size of your site or app and how your entire codebase is set up.

Small to Medium Sites or Apps

I'm going to use my website as the example here. Here is an overview of my files structure:

My Sass is divided up into two major sections: a styles folder, which contains styles that are used throughout the entire site, and individual page Sass files, which contain styles that are specific to only those pages.

The Styles Folder

The styles folder contains the main meat of our Sass. First let's take a look at reset.scss. I include this because browsers tend to try to implement styles for me, so this resets the styling back to zero. This helps out with cross browser styling issues as well. (See this stack overflow article for more information)

I also have created files for the header and footer styling. You guessed it, these contain styling specific to the header and footer, respectively. Not too much to talk about here.

Let's chat about variables. Variables are one of the awesome features of Sass. They allow you to reuse styles in multiple places without having to write the same CSS over and over again. This also helps provide consistency to your styles, just set and forget your base styles, and reuse them everywhere. I created two variable files: color-variables.scss and sizing-variables.scss. These are elements that I will reuse a TON, so they are very key to have variables for. Throughout the rest of my Sass, I no longer write any new colors or sizes, I only use the ones included in here to maintain consistency.
